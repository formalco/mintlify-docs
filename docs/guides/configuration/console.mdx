## Overview

The Formal Console at [app.joinformal.com](https://app.joinformal.com) provides a web-based interface for managing all aspects of your Formal deployment.

## Console Applications

### Resources

[app.joinformal.com/resources](https://app.joinformal.com/resources)

- Create and manage databases, APIs, and infrastructure
- Configure native users and TLS
- Set up schema discovery
- View resource health status

### Connectors

[app.joinformal.com/connectors](https://app.joinformal.com/connectors)

- Deploy and configure Connectors
- Manage listeners and routing rules
- Generate API tokens
- Monitor Connector status

### Users & Groups

- [Users](https://app.joinformal.com/users) - Manage human and machine users
- [Groups](https://app.joinformal.com/groups) - Organize users into teams

### Policies

[app.joinformal.com/policies](https://app.joinformal.com/policies)

- Write and test Rego policies
- Dry-run policies before activation
- Configure notifications
- Manage policy owners

### Logs & Sessions

- [Logs](https://app.joinformal.com/logs) - Search and analyze all queries
- [Sessions](https://app.joinformal.com/sessions) - Monitor active connections

### Integrations

- [Cloud Accounts](https://app.joinformal.com/cloud-accounts) - AWS/GCP/Azure
- [BI Applications](https://app.joinformal.com/bi-integrations) - Metabase, Looker, etc.
- [Log Integrations](https://app.joinformal.com/log-integrations) - S3, Datadog, Splunk
- [SSO](https://app.joinformal.com/sso) - Single sign-on configuration
- [Directory Sync](https://app.joinformal.com/directory-sync) - User/group sync

### Satellites

[app.joinformal.com/satellites](https://app.joinformal.com/satellites)

- Deploy Data Classifiers
- Configure Data Discovery
- Manage Policy Data Loaders

### Spaces

[app.joinformal.com/spaces](https://app.joinformal.com/spaces)

- Create logical groupings
- Segment environments

### API Keys

[app.joinformal.com/api-keys](https://app.joinformal.com/api-keys)

- Generate API keys for Terraform/API access
- Manage key permissions and expiration

## Authentication

### SSO

Configure SSO for your organization:

<Steps>
  <Step title="Navigate to SSO">
    Go to [SSO](https://app.joinformal.com/sso)
  </Step>
  <Step title="Add Connection">Click **Add new SSO connection**</Step>
  <Step title="Select Provider">
    Choose your identity provider (Okta, Azure AD, Google, etc.)
  </Step>
  <Step title="Follow Setup">Complete the WorkOS-guided setup</Step>
  <Step title="Configure Domains">Add allowed email domains</Step>
  <Step title="Enable Self Sign-Up">
    (Optional) Allow users to create accounts automatically
  </Step>
</Steps>

See [SSO Integration](/docs/guides/integrations/sso) for details.

### Directory Sync

Automatically sync users and groups from your identity provider:

<Steps>
  <Step title="Navigate to Directory Sync">
    Go to [Directory Sync](https://app.joinformal.com/directory-sync)
  </Step>
  <Step title="Follow Setup">
    Complete the WorkOS-guided setup for your provider
  </Step>
  <Step title="Configure Sync">Choose which groups and users to sync</Step>
</Steps>

## API Keys

API keys authenticate Terraform, SDKs, and direct API access.

<Steps>
  <Step title="Navigate to API Keys">
    Go to [API Keys](https://app.joinformal.com/api-keys)
  </Step>
  <Step title="Create Key">Click **Create API Key**</Step>
  <Step title="Name Key">Provide a descriptive name</Step>
  <Step title="Copy Key">
    Save the key immediately - it won't be shown again
  </Step>
</Steps>

<Warning>
  API keys are for **administrative access** (managing Formal objects). They are
  **not** the same as user access tokens for connecting through Connectors.
</Warning>

### Using API Keys

<Tabs>
  <Tab title="Terraform">
    ```hcl
    provider "formal" {
      api_key = var.formal_api_key
    }
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -H "Authorization: Bearer YOUR_API_KEY" \
      https://api.joinformal.com/v1/resources
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    from formal_sdk import Formal
    
    formal = Formal(api_key="YOUR_API_KEY")
    ```
  </Tab>
</Tabs>

## User Access Tokens

End-users retrieve their Formal credentials for connecting through Connectors:

<Steps>
  <Step title="Log In">
    Users log in to [app.joinformal.com](https://app.joinformal.com)
  </Step>
  <Step title="Access Token">
    Click **Access Token** button in bottom right
  </Step>
  <Step title="Copy Credentials">Copy Formal username and access token</Step>
  <Step title="Connect">Use credentials to connect through Connectors</Step>
</Steps>

See [Permissions](/docs/guides/policies/permissions) for details.
