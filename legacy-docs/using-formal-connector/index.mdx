# Using Formal Connector

Formal Connector enables secure access to various data sources and technologies. Choose from the supported technologies below for detailed configuration instructions.

## Supported Technologies

- **[PostgreSQL](/using-formal-connector/postgres)** - Connect to PostgreSQL databases
- **[MongoDB](/using-formal-connector/mongodb)** - Connect to MongoDB databases
- **[Snowflake](/using-formal-connector/snowflake)** - Access Snowflake data warehouses
- **[BigQuery](/using-formal-connector/bigquery)** - Connect to Google BigQuery
- **[ClickHouse](/using-formal-connector/clickhouse)** - Access ClickHouse databases
- **[DynamoDB](/using-formal-connector/dynamodb)** - Connect to AWS DynamoDB
- **[S3](/using-formal-connector/s3)** - Access AWS S3 buckets and objects
- **[SSH](/using-formal-connector/ssh)** - Secure shell access to servers and infrastructure
- **[HTTP](/using-formal-connector/http)** - Connect to REST APIs and web services
- **[Kubernetes](/using-formal-connector/kubernetes)** - Access Kubernetes clusters and resources
- **[Smart Routing](/using-formal-connector/smart-routing)** - Automatically route connections to the appropriate resource

For deployment instructions, see [Deploying a Connector](/deploying-a-connector/introduction).
